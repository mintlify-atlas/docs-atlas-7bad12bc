ShedLock is a Java library that prevents your scheduled tasks from running concurrently across multiple application instances. When one node acquires a lock and starts executing a task, all other nodes skip that execution — ensuring exactly-once behavior in distributed environments.

## What ShedLock does

In a horizontally scaled application, every node runs the scheduler. Without coordination, a task scheduled every 15 minutes fires on every node simultaneously. ShedLock solves this by writing a lock record to an external store — the first node to write the record wins, all others skip.

<Note>
ShedLock is **not** a distributed scheduler. It does not queue tasks or delay execution on other nodes — other nodes simply skip the execution. If you need full distributed scheduling, consider [db-scheduler](https://github.com/kagkarlsson/db-scheduler) or [JobRunr](https://www.jobrunr.io/en/).
</Note>

## Get started

<Steps>
  <Step title="Add the dependency">
    Add `shedlock-spring` (or your framework's module) to your Maven or Gradle build.
  </Step>
  <Step title="Enable locking">
    Annotate your Spring configuration with `@EnableSchedulerLock`.
  </Step>
  <Step title="Configure a lock provider">
    Set up a `LockProvider` bean backed by your preferred datastore (JDBC, Redis, MongoDB, etc.).
  </Step>
  <Step title="Annotate your tasks">
    Add `@SchedulerLock(name = "myTask")` to each scheduled method you want to protect.
  </Step>
</Steps>
